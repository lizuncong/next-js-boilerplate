## IDE插件清单
通过`.vscode/extensions.json`约定开发需要安装的插件。IDE打开时，如果检测到未安装这些插件，会自动提示。或者在插件面板搜索栏输入`@recommended`也可以查看项目需要安装的插件清单

## commit规范
通过结合`lefthook`和`commitlint`，规范commit提交信息

## 代码格式化
现代化、全功能 的 ESLint 配置，适合使用 Next.js + Tailwind + TypeScript 的项目，注重代码质量和可访问性。

同时搭配`.vscode/settins.json`使用，效果更佳

## knip
这是一个用于检测项目中未使用代码和依赖的工具

Knip = "Knip Unused"（剪掉未使用的）
- 检测未使用的文件
- 检测未使用的依赖（dependencies / devDependencies）
- 检测未使用的导出（exports）
- 检测未使用的二进制命令

## @lingual/i18n-check
国际化（i18n）代码检查工具，用于在开发阶段发现和修复翻译相关的问题。

| 功能             | 说明                                      |
| -------------- | --------------------------------------- |
| **缺失 key 检测**  | 发现代码中使用了但翻译文件中不存在的 key                  |
| **未使用 key 检测** | 发现翻译文件中存在但代码未使用的 key（死 key）             |
| **类型安全**       | 生成 TypeScript 类型，确保 `t('key')` 有自动补全和校验 |
| **多语言一致性**     | 检查各语言文件的 key 是否对齐                       |
| **命名空间检查**     | 验证命名空间（namespace）使用是否正确                 |
