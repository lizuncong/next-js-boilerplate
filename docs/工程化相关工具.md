## CodeRabbit
CodeRabbit 的定位是“AI 驱动的代码审查助手”，核心作用可以概括为三点：
- 自动 PR 评审
- 节省人工 review 时间
- 低噪声、可定制

## Checkly
Checkly 是一家柏林初创公司提供的「合成监控即代码」平台，主要帮 DevOps / SRE / 前端团队把 API 与 Web 端可用性、性能、正确性 的监控做成 可编程、可 Git 管理、可 CI/CD 集成 的自动化流程，核心能力分三块
- Detect - 监测 & 测试
  + API 检查：从全球 20+ 节点定时调用接口，验证状态码、响应体、延迟，支持 setup/teardown 脚本和灵活断言 。
  + 浏览器检查：基于 Playwright 脚本模拟真实用户点击、填写、导航，可做端到端、视觉回归、关键字断言 。
  + 心跳检查：轻量级 TCP/HTTP ping，用于内部服务保活。全部支持 Monitoring-as-Code：把检查写成 JS/TS 文件，随仓库版本化，CLI 一键部署。
- Communicate - 告警 & 状态页
失败即触发多渠道告警（Slack、PagerDuty、Opsgenie、Webhook），并自动生成公开/私有 Status Page 展示实时可用性与历史 SLA 数据 。
- Resolve - 排障 & 追踪
失败自动录制 Playwright Trace 与 视频回放，可直接在 Web UI 或 trace.playwright.dev 回放每一步 DOM 快照与网络请求，快速定位故障 。
新功能 Checkly Traces 把合成监控与分布式追踪打通，减少手动关联日志的时间

## Crowdin
Crowdin 是一个云端本地化与多语言翻译协作平台，核心作用是把“翻译工作”变成一条可自动化、可版本控制、可多人协作的 DevOps 流水线

## Knip
Knip 的核心作用一句话就能概括：
“用静态分析帮你揪出 JavaScript / TypeScript 项目里所有‘没人要的代码’——包括未使用的依赖包、未引用的文件、以及被导出却从没有被导入的函数/变量/类型。”
具体可拆成 4 类“垃圾”：
- 使用的 npm 依赖 – 装在 package.json 里却从未 import/require
- 未使用的文件 – 项目目录里没有任何入口会引用到它
- 未使用的导出 – 写了 export 却从来没有被其他模块消费
- 未使用的类型/命名空间/重复导出 – 只对类型系统占位置，运行时同样浪费打包体积
