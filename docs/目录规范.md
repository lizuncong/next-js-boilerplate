## 目录规范
遵循下面的目录规范。这里需要注意区分libs和utils目录的区别。

| 原则                  | 说明                   |
| ------------------- | -------------------- |
| **libs 可以依赖 utils** | 但 utils 不应该依赖 libs   |
| **utils 保持纯净**      | 不引入业务逻辑、不访问外部服务      |
| **libs 保持封装**       | 对外暴露简洁 API，隐藏实现细节    |
| **避免循环依赖**          | utils ↔ libs 互相引用是大忌 |

```shell
src/
├── libs/                    # 基础设施层
│   ├── api/                 # HTTP 客户端封装
│   ├── db/                  # 数据库
│   ├── auth/                # 认证
│   ├── i18n/                # 国际化
│   └── logger/              # 日志
│
├── utils/                   # 通用工具层
│   ├── format.ts            # 格式化
│   ├── validate.ts          # 校验
│   ├── cn.ts                # tailwind 类名合并
│   └── sleep.ts             # 延迟函数
│
├── services/                # 业务服务层（可选）
│   ├── user-service.ts
│   └── order-service.ts
│
└── components/              # UI 组件
```
